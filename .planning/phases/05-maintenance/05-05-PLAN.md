<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → /owner/approve/[token] | No auth; UUID token is the credential; one-time use enforced by nullifying on first use |
| Public internet → /owner/decline/[token] | Same — form submit uses server action with admin client |
| Admin client → work_orders update | Service role bypasses RLS; scope limited to the exact row matched by token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-16 | Spoofing | Owner approval token replay | mitigate | Both tokens nullified atomically with status update; `.eq('status', 'pending_approval')` on update prevents double-fire |
| T-05-17 | Elevation of Privilege | Admin client in approve/decline pages | mitigate | Admin client limited to the single work_orders row matching the token; no org-wide queries |
| T-05-18 | Information Disclosure | Work order details in approval email | accept | Email sent to property owner who is the authorized party for this data |
| T-05-19 | Tampering | CSRF on decline form | mitigate | Server action validates token against DB before executing update; no CSRF token needed per Next.js 15 Server Actions (built-in origin check) |
</threat_model>
