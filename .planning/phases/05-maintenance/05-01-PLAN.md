    Create a new Supabase migration file. The migration must:

       - `work_order_priority` with values: 'low', 'medium', 'high', 'urgent'

    2. Create table `work_orders` with columns (per D-15 + schema_push_requirement):
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `org_id UUID NOT NULL REFERENCES organizations(id)`
       - `property_id UUID NOT NULL REFERENCES properties(id)`
       - `unit_id UUID REFERENCES units(id)` (nullable)
       - `title TEXT NOT NULL`
       - `description TEXT NOT NULL`
       - `priority work_order_priority NOT NULL DEFAULT 'medium'`
       - `status work_order_status NOT NULL DEFAULT 'draft'`
       - `assigned_vendor_id UUID REFERENCES people(id)` (nullable)
       - `vendor_token UUID UNIQUE DEFAULT gen_random_uuid()` — pre-generated at insert
       - `estimated_cost NUMERIC(10,2)` (nullable)
       - `vendor_cost NUMERIC(10,2)` (nullable)
       - `billed_amount NUMERIC(10,2)` (nullable)
       - `owner_decline_note TEXT` (nullable)
       - `owner_approve_token UUID UNIQUE` (nullable — set on entering pending_approval)
       - `owner_decline_token UUID UNIQUE` (nullable — set on entering pending_approval)
       - `created_by UUID NOT NULL REFERENCES people(id)`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

    3. Create indexes:
       - `idx_work_orders_org_id` on `(org_id)`
       - `idx_work_orders_property_id` on `(property_id)`
       - `idx_work_orders_status` on `(status)`
       - `idx_work_orders_assigned_vendor` on `(assigned_vendor_id)`
       - `idx_work_orders_vendor_token` on `(vendor_token)`

    4. Enable RLS: `ALTER TABLE work_orders ENABLE ROW LEVEL SECURITY;`

    5. RLS policies:
       - "managers_full_crud": FOR ALL TO authenticated, USING + WITH CHECK that org_id matches caller's org AND 'manager' = ANY(role array from people where user_id = auth.uid())
       - "tenants_insert": FOR INSERT TO authenticated, WITH CHECK org_id matches caller's org AND 'tenant' = ANY(role)
       - "tenants_select_own": FOR SELECT TO authenticated, USING created_by = (SELECT id FROM people WHERE user_id = auth.uid() AND active = true)

    Use `= ANY(SELECT role FROM people WHERE user_id = auth.uid() AND active = true)` pattern for role checks — consistent with existing RLS in the project. Do not use `=` (role is an array type).

    The migration file name must sort after existing migrations — use timestamp `20260623000000`.
