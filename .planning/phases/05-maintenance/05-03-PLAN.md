    **`src/app/(manager)/maintenance/new/page.tsx`** (client component — needs react-hook-form):
    - Form fields: title (text, required), description (textarea, required), priority (select: low/medium/high/urgent, default medium), property_id (select from properties in org), unit_id (optional select, filtered by chosen property).
    - Zod schema for validation (client-side via zodResolver).
    - On submit: calls `createWorkOrder(data)` server action. On success, redirect to `/maintenance`. Show toast error on failure.
    - After creation, work order is in 'draft' status. A manager must explicitly submit it (via the detail page).

    **`src/app/(manager)/maintenance/[id]/page.tsx`** (RSC):
    - Fetch single work order by id with joins: property address, unit number, vendor name/phone/email, creator name.
    - Do NOT select owner_approve_token or owner_decline_token.
    - Sections:
      1. Header: title, status badge, priority badge, created by + date
      2. Details: property + unit, description
      3. Cost fields (manager-only visible): estimated_cost, vendor_cost, billed_amount — display as read-only; these are editable via transition actions
      4. Vendor section: assigned vendor name/contact, or "Unassigned"
      5. Transition buttons section: rendered by `<TransitionButton>` component (see below)

    **`src/components/work-orders/TransitionButton.tsx`** (client component):
    - Props: `{ workOrderId: string, currentStatus: WorkOrderStatus, userRole: string[] }`
    - Uses TRANSITIONS map to determine which next states are available for this role
    - Renders a button per available transition. Button labels:
      - submitted → "Submit Work Order"
      - assigned → show in AssignVendorDialog (Plan 05-04), not here
      - in_progress → "Mark In Progress"
      - completed → "Mark Completed" (shows vendor_cost + billed_amount inputs inline)
      - closed → "Close Work Order"
    - Calls `updateWorkOrderStatus(workOrderId, newStatus, extraData)` on click
    - Shows loading state during transition; displays error toast on failure
    - The "Mark Completed" transition must collect vendor_cost and billed_amount (per D-13/MAINT-08) via inline number inputs before calling the action — these are required for expense auto-creation (MAINT-09)
    - Assignment transition is deliberately excluded here — it is rendered in AssignVendorDialog in Plan 05-04
