<objective>
Create the complete Phase 1 database layer: the `organizations` and `people` tables, the JWT custom-claims Auth Hook, the RLS helper functions, RLS policies for all six roles, org-scoped storage buckets, the plan-limit enforcement trigger, and push the schema to the remote Supabase project.

Purpose: This is the security core. The Auth Hook (claims injected server-side at sign-in) and the `(SELECT auth.org_id())` RLS pattern are the two bets that prevent the most common multi-tenant rewrites. RLS-enabled-without-policies is a silent deny-all, so every table ships with both RLS enabled AND policies in the same migration. Plan-limit enforcement (ORGS-06) is a DB trigger so it is the final gate regardless of UI/Server Action — never UI-only (RESEARCH Architectural Responsibility Map).
Output: Seven migration files applied to the remote DB, plus generated TypeScript Database types.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client JWT → RLS | per-row org isolation enforced at DB layer |
| auth sign-in → Auth Hook | claims set server-side, not client-writable |
| storage upload → bucket | org_id path-prefix isolation |
| unit INSERT → plan limit | enforced at DB trigger, not UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | RLS policies (all tables) | mitigate | `org_id = (SELECT auth.org_id())` on every table; admin cross-org explicit |
| T-02-02 | Elevation of Privilege | JWT app_metadata claims | mitigate | Claims injected by SECURITY DEFINER hook from public.people; not client-writable |
| T-02-03 | Performance/DoS | RLS function evaluation | mitigate | All helper calls wrapped in (SELECT ...) — per-query not per-row (Pitfall 2) |
| T-02-04 | Information Disclosure | storage.objects | mitigate | First path segment must equal caller org_id |
| T-02-05 | Tampering | schema drift via Dashboard | accept | Team rule: migrations only (Pitfall 7); low risk single-developer |
| T-02-06 | Elevation of Privilege | bypassing plan limit via direct INSERT | mitigate | BEFORE INSERT trigger enforce_plan_unit_limit() rejects at DB layer (ORGS-06) |
</threat_model>
