<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| invite email link → account | token must be single-use and pre-scope org/role |
| remove user → sessions | revocation must be immediate, server-side |
| settings form → org row | only managers/admins in-org may edit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | stale JWT after removal | mitigate | admin.signOut(userId,'global') on remove (D-11, Pitfall 4) |
| T-06-02 | Spoofing | invite token reuse | mitigate | accept only where invite_accepted_at IS NULL; mark accepted on use |
| T-06-03 | Information Disclosure | invite/settings actions | mitigate | server-side authz (manager/admin in-org) + RLS org scope |
| T-06-04 | Tampering | Resend API key | mitigate | RESEND_API_KEY read server-only in lib/email/send.ts |
</threat_model>
