# Phase 1 ‚Äî UI Design Contract

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI-managed, initialize before implementation begins) |
| Preset | none ‚Äî greenfield project; run `npx shadcn@latest init` with default New York style |
| Component library | Radix UI primitives (via shadcn/ui) |
| Icon library | Lucide React (ships with shadcn/ui) |
| Font | Inter (system-ui fallback chain: `Inter, system-ui, -apple-system, sans-serif`) |

**shadcn/ui initialization note:** No `components.json` exists. Before any component implementation, run:

```bash
npx shadcn@latest init
```

Select: New York style, neutral base color, CSS variables enabled, TypeScript. This produces `components.json` and injects CSS custom properties into `src/app/globals.css`. All subsequent shadcn components are added via `npx shadcn@latest add {component}`.

Tailwind v4 CSS-first config applies ‚Äî no `tailwind.config.js` required.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline micro-spacing |
| sm | 8px | Input internal padding (vertical), compact list item gaps |
| md | 16px | Default element spacing, form field gaps, card padding (horizontal) |
| lg | 24px | Section padding, card padding (vertical), wizard step gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Auth card max-width inner padding, major section breaks |
| 3xl | 64px | Page-level top/bottom padding on desktop |

Exceptions:
- Touch targets on mobile: minimum 44px height for all interactive elements (buttons, links, form controls) ‚Äî enforced via `min-h-11` (44px) in Tailwind v4
- Wizard step indicator circles: 32px diameter
- Avatar/logo upload area: 80px √ó 80px
- OAuth provider button height: 44px (touch-safe)

|------|------|--------|-------------|-------|

## Color

The Canary brand is warm and trustworthy. The palette uses a warm amber/yellow as accent against a clean neutral surface ‚Äî professional property management, not cold fintech.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF9` (warm white, stone-50 equivalent) | Page background, auth card background, portal shell content area |
| Secondary (30%) | `#F5F4F2` (warm gray, stone-100 equivalent) | Sidebar background, nav rail, wizard step panel, card backgrounds, input surfaces |
| Accent (10%) | `#D97706` (amber-600) | Reserved elements listed below |
| Destructive | `#DC2626` (red-600) | Remove user from org button, session revoke confirmation, destructive modal confirm button only |

**Accent (`#D97706`) reserved for exactly these elements:**
1. Primary CTA buttons (Sign In, Continue, Create Organization, Send Invite)
2. Active/selected wizard step indicator circle
3. "Complete your setup" banner left border stripe
4. Invite status badge background for "Active" state
5. Focused form field ring (replaces default blue focus ring)

**Accent is NOT used on:** navigation links, secondary buttons, status text, error states, helper text, or any decorative purpose.

**Semantic colors for status badges:**
- `Invite sent` (pending): `#92400E` text on `#FEF3C7` background (amber-800 / amber-100)
- `Active`: `#065F46` text on `#D1FAE5` background (emerald-800 / emerald-100)

**Dark mode:** Not in scope for Phase 1. shadcn/ui's CSS custom properties will make this trivial to add later. Do not add dark mode toggling now.

## Screen-by-Screen Contracts

### 1. Sign-In Page (`app.canarypm.ca/login`)

**Layout:** Centered card, max-width 400px, vertically centered on viewport. Full-bleed warm white background. Canary wordmark top-center of card.

**Interaction flow:**
1. Email field (autofocus on mount)
2. Password field (show/hide toggle via `Eye` / `EyeOff` Lucide icon, 20px)
3. "Sign in" primary button (full width, amber accent, 44px height)
4. Separator: "Or"
5. "Continue with Google" button ‚Äî white background, 1px stone-300 border, Google SVG logo (20px), 44px height, full width
6. "Continue with Apple" button ‚Äî black background, white text, Apple SVG logo (20px), 44px height, full width
7. "Send me a magic link" text link below OAuth buttons (14px, accent color)
8. "Don't have an account? Create one" link at card bottom

**Magic link flow:** Clicking "Send me a magic link" replaces the password field and Sign in button with: email-only input + "Send magic link" button. On submit, show inline confirmation: "Check your email ‚Äî we've sent a sign-in link to {email}."

**Loading states:**
- Sign in button: spinner replaces button text during auth request. Button disabled. Do not show skeleton loaders.
- OAuth redirect: immediately show "Redirecting to Google..." / "Redirecting to Apple..." inline below the clicked button, 14px muted text. Button disabled.

**Post-auth redirect (D-04):**
- `role=manager` or `role=employee` ‚Üí `/dashboard`
- `role=tenant` ‚Üí `/my-home`
- `role=owner` ‚Üí `/portfolio`
- `role=vendor` ‚Üí `/jobs`
- `role=admin` ‚Üí `/admin`

### 2. Org Sign-Up / Onboarding Wizard

**Layout:** Centered card, max-width 480px. Step indicator at top of card. Step counter "Step N of 5" in 14px label/caption. `Progress` bar beneath counter showing percentage complete.

**Step 1 ‚Äî Org Name (required)**
- Heading: "What's your company called?"
- Body: "This is how your organization will appear to managers, tenants, and owners."
- Input: `Organization name` label, placeholder "e.g. Canary Property Management"
- Validation: required, min 2 chars, max 80 chars. Inline error beneath field on blur.
- CTA: "Continue" (primary, amber, full width)

**Step 2 ‚Äî Logo Upload (optional)**
- Heading: "Add your logo"
- Body: "Your logo appears on tenant portals and email communications."
- Upload target: 80√ó80px avatar with `Upload` Lucide icon centered. Click or drag to upload. Accept: image/png, image/jpeg, image/webp. Max 2MB.
- On upload: show preview in 80√ó80 avatar circle with "Remove" text link below.
- CTA row: "Continue" (primary) + "Skip for now" (ghost/link style, 14px)

**Step 3 ‚Äî Province/Jurisdiction (required)**
- Heading: "Where do you operate?"
- Body: "Required for Canadian tenancy compliance rules."
- `Select` component: label "Province or Territory". Options: all 13 Canadian provinces and territories in alphabetical order (Alberta, British Columbia, Manitoba, New Brunswick, Newfoundland and Labrador, Northwest Territories, Nova Scotia, Nunavut, Ontario, Prince Edward Island, Quebec, Saskatchewan, Yukon).
- Validation: required. Inline error if Continue clicked with no selection.
- CTA: "Continue" (primary)

**Step 4 ‚Äî Invite First Manager (optional)**
- Heading: "Invite your first team member"
- Body: "They'll receive an email to join your organization as a manager."
- Input: `Email address` label, placeholder "colleague@yourcompany.com"
- Validation: valid email format only. Optional field ‚Äî empty is allowed.
- CTA row: "Send invite & continue" (primary) + "Skip for now" (ghost/link style, 14px)

**Step 5 ‚Äî Dashboard (completion)**
- Heading: "You're all set, {org name}!"
- Body: "Your workspace is ready. Let's add your first property."
- CTA: "Go to dashboard" (primary)
- This step is the redirect target after all previous steps complete.

**Skipped steps ‚Äî persistent setup banner (D-02):**
- `Alert` component, full width, positioned below top nav on dashboard
- Left border: 4px solid amber-600 accent stripe
- Icon: `AlertCircle` Lucide (amber-600, 20px)
- Copy: "Complete your setup ‚Äî {N item(s)} remaining. [Continue setup ‚Üí]"
- Dismiss: `X` icon (16px) at right. Dismissal persists in localStorage. Banner reappears if user visits `/settings/setup`.

### 3. Portal Shells (empty scaffolding)

Four role-specific layout shells. Each has the same structural pattern with role-appropriate navigation labels.

**Shell structure (all roles):**
- Mobile: bottom tab bar (4‚Äì5 items) + hamburger for secondary nav
- Desktop: left sidebar (240px fixed width) + main content area (fluid)
- Top bar: org logo (32px) + org name (16px semibold) at sidebar top; user avatar + name + dropdown at sidebar bottom
- Content area: 16px padding on mobile, 32px padding on desktop, max-width 1280px centered

**Manager shell nav items:**
Dashboard, Properties, People, Leases, Maintenance, Payments, Settings

**Tenant shell nav items:**
Home, Pay Rent, Maintenance, My Lease

**Owner shell nav items:**
Portfolio, Statements, Maintenance

**Vendor shell nav items:**
My Jobs

**Empty state within each shell:**
- Each nav section lands on an empty content area
- Heading: "{Feature name} coming soon"
- Body: "This section is being built. Check back in the next update."
- No illustration ‚Äî plain text only in Phase 1 shells.

### 4. People List (Manager ‚Äî Invite Status)

**Layout:** Table or stacked card list (mobile: cards; desktop: table). Shows: Name, Email, Role, Status, Actions.

**Status badge (D-09):**
- `Invite sent`: amber-100 background, amber-800 text, `Clock` Lucide icon (14px) inline-left, "Invite sent" text
- `Active`: emerald-100 background, emerald-800 text, `CheckCircle2` Lucide icon (14px) inline-left, "Active" text

**Empty state (no people invited yet):**
- Heading: "No team members yet"
- Body: "Invite a manager or employee to get started."
- CTA: "Invite someone" (primary button, amber accent)

### 5. Org Settings Page

**Layout:** Single-column form, max-width 640px, 32px section gaps.

**Sections:**
1. Organization name (Input, save on button click ‚Äî not auto-save)
2. Logo (Avatar upload, same pattern as onboarding Step 2)
3. Province/Territory (Select, same options as onboarding Step 3)
4. Branding color ‚Äî placeholder section with "Coming soon" label (Phase 11 feature, render disabled input)

CTA per section: "Save changes" (primary button, right-aligned).

Success feedback: Sonner toast ‚Äî "Changes saved" (bottom-right, 3s auto-dismiss).

### 6. Tenant Invite Email Template

**Platform:** react-email component rendered via Resend.

**Layout:** Single-column, max-width 600px, centered, white background, 32px padding.

**Header:** Canary wordmark (text-based if no logo uploaded, org logo if available).

**Body copy (D-07):**
```
Hi {tenant_first_name},

{Org name} has invited you to manage your tenancy at:

  {property_address}
  Unit {unit_number}
  Move-in date: {move_in_date, formatted as "January 15, 2025"}

Click the button below to create your account and access your tenant portal.
```

**CTA button:** "Set up my account" ‚Äî amber-600 background (#D97706), white text, 16px font, 48px height, 24px horizontal padding, 6px border radius.

**Footer:** 14px gray text ‚Äî "This invite was sent by {org_name}. If you didn't expect this email, you can ignore it."

## Error States & Copy

All error copy follows: **[What went wrong] + [What to do next].** No raw error codes, no technical terms.

| Error | Heading | Body |
|-------|---------|------|
| Invalid email/password | "Couldn't sign you in" | "The email or password you entered doesn't match our records. Try again or send a magic link." |
| Expired magic link | "This link has expired" | "Magic links are valid for 1 hour. Enter your email below to get a new one." |
| Revoked session (401 on API) | "You've been signed out" | "Your access was removed by an administrator. Contact your property manager if you think this is an error." |
| OAuth provider error | "Sign-in didn't complete" | "Something went wrong connecting to {Google / Apple}. Try again or sign in with email." |
| Invite link already used | "This invite has already been accepted" | "If you already have an account, sign in at app.canarypm.ca/login." |
| Invite link expired | "This invite has expired" | "Ask your property manager to send a new invite." |
| Org at unit limit (free plan) | "Unit limit reached" | "Your free plan includes up to 5 units. Upgrade to add more properties." |
| Network error | "Connection problem" | "Check your internet connection and try again." |

## Accessibility Contracts

- All form inputs have visible `Label` components associated via `htmlFor`/`id` ‚Äî never aria-label alone
- All icon-only buttons have `aria-label` and `title` attributes (e.g., password show/hide toggle)
- Focus ring: amber-600 at 2px offset (replaces default browser blue) ‚Äî do not remove focus rings
- Color is never the sole indicator of status ‚Äî badges always include icon + text, not color alone
- Touch targets: minimum 44√ó44px for all interactive elements
- Province `Select` is keyboard-navigable via Radix UI primitives (shadcn Select handles this)
- Auth form submits on Enter key from any field
- Wizard "Skip for now" links are keyboard-reachable and visually distinguishable from body text
