URL filter pattern (LIST-02) — use searchParams for server-side filtering:
  /listings?minRent=1000&maxRent=2000&beds=2&amenity=parking
  Page receives { searchParams: Promise<{ minRent?: string, maxRent?: string, beds?: string, amenity?: string }> }

Query joining listings with units and properties (all via Supabase PostgREST):
  supabase.from('listings')
    .select('id, listing_title, display_rent, highlights, available_from, status,
             units!unit_id(id, bedrooms, bathrooms, sq_footage, asking_rent, amenities,
               properties!property_id(id, street_address, city, province, photo_paths))')
    .eq('status', 'published')
    .eq('org_id', orgId)

Google Maps embed (D-08-map):
  src={`https://www.google.com/maps/embed/v1/place?key=${process.env.NEXT_PUBLIC_GOOGLE_MAPS_KEY}&q=${encodeURIComponent(address)}`}
  where address = `${street_address}, ${city}, ${province}`
  width="100%" height="300" style={{ border: 0 }} allowFullScreen loading="lazy"

Supabase signed URL for property photos:
  Photos are stored in org-assets bucket as photo_paths (array of storage paths).
  For public listing pages, use the public URL pattern:
  supabase.storage.from('org-assets').getPublicUrl(path).data.publicUrl
  (org-assets bucket was set to public in Phase 1; if private, use createSignedUrls instead)
