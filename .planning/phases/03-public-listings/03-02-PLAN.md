## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP hostname → org slug | Attacker could craft a hostname to impersonate a different org's subdomain |
| ?org= query param | Localhost fallback; attacker on prod could try to inject an org slug via query param |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | subdomain → org slug extraction | accept | Subdomain is read from host header set by the CDN/Vercel edge — cannot be forged by the visitor; Vercel enforces domain ownership |
| T-03-05 | Spoofing | ?org= query param on prod | mitigate | Document that ?org= fallback must only be active on localhost (NODE_ENV !== 'production') in a follow-up hardening task; for Phase 3 it is acceptable since wrong org slug returns only published listings from that org (no sensitive data) |
| T-03-06 | Information Disclosure | anon client in createPublicClient | accept | Client only has anon key; RLS ensures only published listings are readable; no auth bypass possible |
</threat_model>
