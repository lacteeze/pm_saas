Gmail API (googleapis package):
```typescript
import { google } from 'googleapis'

// OAuth2 client setup:
const oauth2Client = new google.auth.OAuth2(
  process.env.GMAIL_CLIENT_ID,
  process.env.GMAIL_CLIENT_SECRET,
  process.env.GMAIL_REDIRECT_URI  // must be https://yourdomain.com/api/gmail/callback
)

// Generate auth URL:
const authUrl = oauth2Client.generateAuthUrl({
  access_type: 'offline',
  scope: ['https://mail.google.com/'],
  prompt: 'consent',  // force refresh_token on every connect
  state: orgId,       // pass orgId through state param
})

// Exchange code for tokens:
const { tokens } = await oauth2Client.getToken(code)
// tokens.access_token, tokens.refresh_token, tokens.expiry_date (ms)

// Refresh check:
if (Date.now() > tokenExpiry - 60_000) {
  oauth2Client.setCredentials({ refresh_token: storedRefreshToken })
  const { credentials } = await oauth2Client.refreshAccessToken()
  // update DB with new credentials.access_token + credentials.expiry_date
}

// Search Gmail:
const gmail = google.gmail({ version: 'v1', auth: oauth2Client })
const response = await gmail.users.messages.list({
  userId: 'me',
  q: 'from:notifications@payments.interac.ca subject:"INTERAC e-Transfer"',
  maxResults: 50,
})
// For each message, fetch full body: gmail.users.messages.get({ userId: 'me', id: msg.id, format: 'full' })
// Parse amount: /\$([0-9,]+\.[0-9]{2})/ from snippet or body
// Parse sender name: from the email body text (varies by Interac format)
```

Interac e-transfer parse targets (from research):
- Amount pattern: `\$([0-9,]+\.[0-9]{2})`
- Sender name: usually in subject line "John Smith sent you $500.00 via INTERAC e-Transfer"
  Parse: subject.match(/^(.+?) sent you/)
- Date: use message internalDate (unix ms)

Create `src/lib/gmail.ts` exporting four functions. All are server-only (no 'use client').

**`getGmailAuthUrl(orgId: string): string`**
Creates an OAuth2Client with GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET, GMAIL_REDIRECT_URI env vars. Calls generateAuthUrl with access_type='offline', scope=['https://mail.google.com/'], prompt='consent', state=orgId. Returns the URL string.

**`exchangeCodeForTokens(code: string): Promise<{ access_token: string, refresh_token: string, expiry_date: number }>`**
Creates OAuth2Client, calls getToken(code), returns the token fields. Throws if refresh_token is missing (user must reconnect).

**`refreshTokenIfNeeded(orgId: string, supabase: SupabaseClient): Promise<string>`**
Fetches org row: SELECT gmail_access_token, gmail_refresh_token, gmail_token_expiry. If access_token is null, throws 'Gmail not connected'. If Date.now() > gmail_token_expiry - 60_000: creates OAuth2Client, sets credentials with refresh_token, calls refreshAccessToken(), updates DB row (gmail_access_token, gmail_token_expiry), returns new access_token. Otherwise returns current gmail_access_token.

**`searchETransfers(accessToken: string): Promise<Array<{ messageId: string, senderName: string, amount: number, receivedAt: string, subject: string }>>`**
Creates OAuth2Client, sets credentials({ access_token: accessToken }). Calls gmail.users.messages.list with q='from:notifications@payments.interac.ca subject:"INTERAC e-Transfer"' and maxResults=50. For each message, calls messages.get with format='metadata', metadataHeaders=['Subject','Date']. Parses senderName from subject using `/^(.+?) sent you/i`. Parses amount using `/\$([0-9,]+\.[0-9]{2})/`. Returns array — skips messages where parsing fails. Uses internalDate for receivedAt (convert from ms to ISO string).

Use the `googleapis` package. Add to package.json if not present (run `npm install googleapis` from canary-propos/).

**`src/app/api/gmail/callback/route.ts`**

Export `GET`. Route handler receives ?code=...&state=orgId from Google OAuth redirect.

Steps:
1. Extract code and state (orgId) from URL params — return 400 if missing
2. Call exchangeCodeForTokens(code) — return 400 if it throws
3. Get supabase = createAdminClient() (service role — needed to write to organizations without RLS)
4. UPDATE organizations SET gmail_access_token, gmail_refresh_token, gmail_token_expiry, gmail_connected_at=NOW() WHERE id=orgId
5. Return redirect to /settings?gmail=connected

Error case: redirect to /settings?gmail=error on any failure.

**`src/app/api/gmail/etransfers/route.ts`**

Export `GET`. Authenticated — requires manager session.

Steps:
1. Get caller context (createClient, check person.role.includes('manager') or 'admin') — return 401 if not authorized
2. Call refreshTokenIfNeeded(ctx.person.org_id, ctx.supabase) to get a valid access token
3. Call searchETransfers(accessToken)
4. Return JSON array of suggestions — each suggestion: `{ messageId, senderName, amount, receivedAt, subject }`
5. No auto-matching logic — just return raw suggestions. The manager UI (Plan 04-04) will handle the confirm flow.

Return 200 with empty array if Gmail not connected or no messages found. Return 500 with error message on unexpected failure.

| Boundary | Description |
|----------|-------------|
| Google OAuth → /api/gmail/callback | Unauthenticated GET with code — state param carries orgId, must be validated |
| manager → /api/gmail/etransfers | Authenticated — only org's own Gmail tokens are used |
| organizations.gmail_* columns | Token storage — admin client writes; regular client reads via RLS |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-09 | Information Disclosure | gmail_refresh_token | mitigate | Never returned to client; stays server-side only; no SELECT in non-admin queries |
| T-04-10 | Spoofing | /api/gmail/etransfers | mitigate | Requires manager session; uses org's own stored tokens only |
