Disbursement formula (from D-07 + research):
```typescript
// management_fee_type and management_fee_value from properties table
// Both columns nullable — treat null as 0 fee

function calculateManagementFee(
  rentCollected: number,
  feeType: 'percent' | 'flat' | null,
  feeValue: number | null
): number {
  if (!feeType || feeValue === null) return 0
  if (feeType === 'percent') return rentCollected * (feeValue / 100)
  return feeValue // flat
}

// net_to_owner = rentCollected - totalBilledExpenses - managementFee
// NOTE: totalBilledExpenses uses billed_amount — never vendor_cost
```

```typescript
export interface DisbursementSummary {
  propertyAddress: string
  ownerName: string
  orgName: string
  rentCollected: number
  expenses: Array<{ id: string; description: string; billedAmount: number }>
  managementFeeType: 'percent' | 'flat' | null
  managementFeeValue: number | null
  managementFee: number
  managementFeeLabel: string
  netToOwner: number
  statementAlreadyExists: boolean
}
```

Steps:
1. getCallerContext() — verify manager/admin
2. Verify property belongs to caller's org
3. Compute date range: first day of month to last day of month (using year + month)
4. Load cleared payments for this property's leases in range: SELECT SUM(amount) FROM payments WHERE lease_id IN (SELECT id FROM leases WHERE property_id=X) AND status='cleared' AND created_at BETWEEN start AND end
5. Load expenses for property in range: SELECT id, description, billed_amount FROM expenses WHERE property_id=X AND expense_date BETWEEN start AND end — NO vendor_cost in this SELECT
6. Load property: SELECT address, management_fee_type, management_fee_value, owner_id
7. Load owner name: SELECT first_name, last_name FROM people WHERE id=owner_id
8. Calculate managementFee using calculateManagementFee helper
9. Build managementFeeLabel: percent → "Management Fee (X%)", flat → "Management Fee (flat $X)", null → "Management Fee"
10. Check if statement already exists: SELECT id FROM owner_statements WHERE property_id=X AND period_year=year AND period_month=month
11. Return DisbursementSummary

**`src/app/api/statements/generate/route.ts`**

Export `POST`. Authenticated — requires manager session.

Request body (Zod validate): `{ property_id: string, year: number, month: number }`

Steps:
1. Get caller context, verify manager/admin role
2. Call calculateDisbursement(property_id, year, month) — return 400 on error
3. If summary.statementAlreadyExists: return JSON `{ error: 'Statement already exists for this period' }` with status 409 — do not regenerate
4. Build StatementData from DisbursementSummary (map expenses to { description, billedAmount })
5. `const pdfBuffer = await renderToBuffer(React.createElement(StatementPDF, { statement: statementData }))` — server-side only
6. Get `supabase = createAdminClient()`
7. Build storage path: `statements/${property_id}/${year}-${String(month).padStart(2,'0')}.pdf`
8. Upload: `supabase.storage.from('org-assets').upload(path, pdfBuffer, { contentType: 'application/pdf', upsert: false })`
9. If upload error (file exists): return 409 — do not insert into owner_statements
10. INSERT into owner_statements: { org_id, property_id, period_year: year, period_month: month, pdf_path: path, rent_collected, total_expenses: sum of billedAmounts, management_fee, net_to_owner, generated_by: person.id }
11. Return JSON `{ success: true, pdf_path: path }`

Mark the route with `export const runtime = 'nodejs'` (renderToBuffer requires Node.js runtime, not Edge).
