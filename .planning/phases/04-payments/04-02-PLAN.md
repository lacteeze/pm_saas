Export `POST` as a Next.js route handler. This route MUST disable body parsing to get the raw body.

At the top of the file add:
```typescript
export const runtime = 'nodejs'
```

Handler steps (in this exact order — order is critical for idempotency):
1. Read raw body: `const rawBody = await req.text()` — NEVER use req.json()
2. Get Stripe-Signature header: `const sig = req.headers.get('stripe-signature') ?? ''`
3. Verify signature: `stripe.webhooks.constructEvent(rawBody, sig, STRIPE_WEBHOOK_SECRET)` — return 400 on failure
4. Get `supabase = createAdminClient()` (service role, bypasses RLS)
5. **Idempotency check FIRST:** `SELECT id FROM stripe_events WHERE stripe_event_id = event.id` — if found, return 200 immediately with no DB writes
6. Process event type:
   - `payment_intent.processing`: INSERT into payments with status='pending_clearance', method='stripe', stripe_payment_intent_id=pi.id, lease_id from pi.metadata.lease_id, org_id from pi.metadata.org_id, amount from pi.amount/100
   - `payment_intent.succeeded`: UPDATE payments SET status='cleared', cleared_at=NOW(), disbursable_after=addBusinessDays(new Date(), 5) WHERE stripe_payment_intent_id=pi.id
   - `payment_intent.payment_failed`: UPDATE payments SET status='failed' WHERE stripe_payment_intent_id=pi.id
7. INSERT into stripe_events: `{ stripe_event_id: event.id, event_type: event.type, payload: event as any }`
8. Return 200
