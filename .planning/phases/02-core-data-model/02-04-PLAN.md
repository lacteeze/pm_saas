<!-- Critical patterns for this plan -->

Lease list query (full join for display):
  supabase.from('leases').select(`
    id, start_date, end_date, monthly_rent, status, renewal_status,
    people!tenant_id(first_name, last_name),
    units!unit_id(unit_number, properties!property_id(street_address, city))
  `).eq('org_id', callerPerson.org_id).order('end_date', { ascending: true })

Expiry query (90-day window):
  const ninetyDaysFromNow = new Date(); ninetyDaysFromNow.setDate(ninetyDaysFromNow.getDate() + 90)
  .eq('status', 'active').lte('end_date', ninetyDaysFromNow.toISOString().split('T')[0])

Days-until-expiry computation in component:
  Math.ceil((new Date(endDate).getTime() - new Date().getTime()) / (1000 * 60 * 60 * 24))

Cascade property→unit select (client component pattern):
  const [selectedPropertyId, setSelectedPropertyId] = useState('')
  const [units, setUnits] = useState([])
  useEffect(() => {
    if (!selectedPropertyId) return
    const supabase = createBrowserClient(...)
    supabase.from('units').select('id, unit_number').eq('property_id', selectedPropertyId).eq('status', 'vacant')
      .then(({ data }) => setUnits(data ?? []))
  }, [selectedPropertyId])

Lease document storage path: {org_id}/leases/{lease_id}/{filename}
Signed URL for tenant download (60-second expiry — security critical per UI-SPEC):
  supabase.storage.from('org-assets').createSignedUrl(lease.document_path, 60)
  Called SERVER-SIDE only (not from client). Returns signedUrl string or null.

Lease status computation (not stored — derived from end_date at render time):
  const today = new Date().toISOString().split('T')[0]
  const daysLeft = Math.ceil((new Date(endDate).getTime() - new Date().getTime()) / 86400000)
  status = lease.status === 'expired' || endDate < today ? 'Expired'
         : daysLeft <= 90 ? 'Expiring'
         : 'Active'
  (Note: DB status field starts as 'active'; display-time 'Expiring' is derived — we do NOT update DB status to 'expiring')

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generateLeaseDownloadUrl | Called server-side; RLS on leases table controls who can fetch document_path |
| Signed URL (60s) | Short-lived URL is the access control mechanism for tenant PDF download |
| LeaseDocUpload → storage | Browser uploads directly; storage RLS enforces org_id path prefix |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Information Disclosure | generateLeaseDownloadUrl called by tenant | mitigate | No explicit role check; RLS on leases (leases_select_tenant) only returns tenant's own lease; non-matching leaseId returns null |
| T-02-14 | Information Disclosure | 60-second signed URL | mitigate | Signed URL is time-limited; cannot be reused after expiry; opened in new tab immediately |
| T-02-15 | Tampering | createLease with unit from different org | mitigate | createLease verifies unit.org_id == callerPerson.org_id before insert |
| T-02-16 | Information Disclosure | Tenant unit select in AddLeaseForm | accept | Tenant does not use AddLeaseForm — this is a manager-only dialog; browser client uses authenticated session |
| T-02-SC | Tampering | npm installs | accept | No new packages in this plan |
