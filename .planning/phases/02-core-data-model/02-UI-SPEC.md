> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal gap |
| sm | 8px | Compact element spacing, mobile card internal gaps |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps, between-section spacing |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Empty state full-bleed vertical padding (py-16) |

Exceptions: Touch targets minimum 44px height (nav items use `min-h-11` = 44px; form inputs use `min-h-10` = 40px, acceptable for non-primary touch targets).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label / meta | 12px (text-xs) | 400 (regular) | 1.4 |
| Heading / section title | 16px (text-base) or 20px (text-xl) | 600 (semibold) | 1.2 |
| Page title | 20px (text-xl) | 600 (semibold) | 1.2 |

Source: Established in people/page.tsx — `text-xl font-semibold` for page title, `text-sm` for body, `text-xs` for table headers.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FAFAF9 (stone-50) | Page background, table row hover |
| Secondary (30%) | #F5F4F2 (stone-100 approx.) | Sidebar, nav, card backgrounds, table header bg |
| Accent (10%) | #D97706 (amber-600) | Active mobile nav icon + label only |
| Destructive | #DC2626 (red-600) | Destructive action text buttons (Remove, Deactivate) |

Accent reserved for: active mobile tab bar icon + label, active state indicator on bottom nav. No other elements use amber-600.

Status badge color mapping (new for Phase 2):

| Status | Background | Text | Usage |
|--------|-----------|------|-------|
| Vacant | green-100 | green-700 | Unit and lease status |
| Occupied | amber-100 | amber-700 | Unit status |
| Maintenance | red-100 | red-700 | Unit status — taken offline for repairs |
| Active (lease) | green-100 | green-700 | Lease status |
| Expiring (lease) | amber-100 | amber-700 | Lease within 90 days of end |
| Expired (lease) | stone-100 | stone-600 | Past lease end date |
| Renewal pending | blue-100 | blue-700 | LEASE-04 renewal_status = pending/sent |

These are inline Tailwind classes applied directly on the Badge component — no new variant needed. Pattern: `className="bg-green-100 text-green-700"` overrides the default variant.

Expiry urgency color in alert callouts:

| Days to expiry | Alert background | Border |
|---------------|-----------------|--------|
| 30 days or fewer | red-50 | red-200 |
| 31–60 days | amber-50 | amber-200 |
| 61–90 days | stone-50 | stone-200 |

## Page-by-Page Layout Specifications

### 1. /people — Two-Tab Layout (D-04)

**Layout:** Existing page header (title + subtitle + CTA) preserved. Below header, add a tab strip using shadcn `Tabs` component.

```
[People]                          [Add Contact ▾]
Manage your team and contacts.

[Team] [Contacts]
──────────────────────────────────────
{tab content}
```

**Team tab:** Current people list as-is (desktop table + mobile cards). No change to existing content.

**Contacts tab:** Same desktop table + mobile card pattern, different columns.

Desktop table columns (Contacts):
- Name (font-medium text-stone-900)
- Email (text-stone-600)
- Phone (text-stone-600)
- Role(s) (comma-separated badges, e.g. "Tenant", "Owner")
- Actions (Edit / Deactivate button)

Mobile card (Contacts):
- Name (font-medium text-stone-900) + Role badge(s) top-right
- Email (text-sm text-stone-500)
- Phone (text-sm text-stone-500)
- Edit link (text-sm text-amber-600)

**Add Contact button:** Top-right of page header, same position as "Invite member". Opens a Dialog modal (not a drawer — consistent with InviteUserForm pattern).

**Add Contact form fields (Dialog):**
- First name (text input, required)
- Last name (text input, required)
- Email (email input, required)
- Phone (tel input, optional)
- Role(s) (multi-select or checkbox group: Tenant / Owner / Vendor — required, at least one)
- Notes (textarea, optional, 3 rows)
- [Cancel] [Add Contact] buttons

**Edit Contact:** Opens same Dialog pre-filled. Deactivate is a red text button in the actions column ("Deactivate") with a confirmation Dialog ("Deactivate {Name}? They will no longer appear in active contact lists.").

**Empty state — Contacts tab:**
```
[dashed border rounded-xl, py-16 text-center]
No contacts yet
Add tenants, owners, and vendors to build your contact directory.
[Add Contact] (button)
```

**Desktop leases table columns:**
- Tenant (font-medium text-stone-900, full name)
- Property + Unit (text-stone-600, "123 Main St — Unit 1A")

- End (text-stone-600, "Aug 31, 2025")
- Monthly Rent (text-stone-600, "$1,400")
- Status (Badge — Active / Expiring / Expired)
- Renewal (Badge — "Pending" / "Accepted" / "Declined" / "—")
- Actions (View)

**Mobile lease card:**
- Tenant name (font-medium) + Status badge (top-right)
- Property + unit (text-sm text-stone-500)

**Add Lease form (Dialog):**

- Unit (select filtered by chosen property, required)
- Lease start (date input, required)
- Lease end (date input, required)
- Monthly rent (currency input, required)
- Deposit amount (currency input, required)
- Rent due day (number select 1–28, default 1, required)
- Lease document (file input, PDF only, optional — LEASE-05)
- [Cancel] [Create Lease] buttons

Note on property → unit cascade: When property changes, unit select resets and repopulates. This requires a client component with controlled state, not a plain RSC form.

**Empty state — leases page:**
```

[Add Lease] (button)
```

### ExpiryAlertCallout

A reusable component used on /leases, /properties/[id] (leases tab), and /dashboard.

Props: `leases: Array<{ id, tenantName, propertyUnit, endDate, daysUntilExpiry }>`

Renders grouped by urgency bucket (≤30, 31–60, 61–90). Each bucket only renders if it has items.

Structure:
```tsx
<div className="rounded-xl border {border-color} {bg-color} p-4 mb-4">
  <div className="flex items-center gap-2 mb-2">
    <AlertTriangle size={16} className="{text-color}" />
    <span className="text-sm font-semibold {text-color}">{label}</span>
  </div>
  <ul className="space-y-1">
    {leases.map(lease => (
      <li className="flex items-center justify-between text-sm">
        <span className="text-stone-700">{tenantName} — {propertyUnit}</span>
        <Link href={`/leases/${id}`} className="text-stone-500 hover:text-stone-900">
          {daysUntilExpiry} days · View →
        </Link>
      </li>
    ))}
  </ul>
</div>
```

### Form Pattern — Add/Edit Dialogs

All create/edit forms use Dialog (shadcn). Pattern:
- Trigger button: top-right of page header, `variant="default"` (filled, black)
- Dialog width: `max-w-md` (single-column form), `max-w-lg` for multi-field forms (Add Property, Add Lease)
- Form grid: single column, `space-y-4` between fields
- Field label: `text-sm font-medium text-stone-700` (shadcn Label)
- Input: shadcn Input, full width
- Required fields: no asterisk; validation error shown below input in `text-xs text-red-600`
- Footer: `flex justify-end gap-2` — [Cancel (outline)] [Submit (default)]
- Loading state: submit button shows "Saving…" text, disabled during Server Action
- Success: Dialog closes, `toast.success("{Record} created")` via Sonner
- Error: `toast.error("Something went wrong. Please try again.")` via Sonner — Dialog stays open
