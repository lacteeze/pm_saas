## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Supabase Storage | Photo upload goes directly from browser to storage; RLS policy enforces org_id path prefix |
| createUnit → plan_unit_limit | Pre-check is UX only; DB trigger is authoritative gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Elevation of Privilege | createProperty/createUnit with wrong org_id | mitigate | getCallerContext() derives org_id from authenticated people row; form data cannot override it |
| T-02-10 | Information Disclosure | /properties/[id] accessing another org's property | mitigate | property fetch includes .eq('org_id', callerPerson.org_id); RLS also enforces |
| T-02-11 | Tampering | PropertyPhotoUpload writing to another org's path | mitigate | Storage RLS: first path segment must match (SELECT public.org_id())::text; browser client uses auth session |
| T-02-12 | Denial of Service | Photo upload bypassing size limit | mitigate | Bucket file_size_limit = 20971520 (20MB) enforced by Supabase Storage on upload |
| T-02-SC | Tampering | npm installs | accept | No new packages; @supabase/ssr createBrowserClient already installed in Phase 1 |
</threat_model>
