<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tenant RSC → leases table | leases_select_tenant RLS: only rows where tenant_id = (SELECT public.person_id()) are returned |
| Tenant → generateLeaseDownloadUrl | Server action fetches document_path; only accessible if RLS returns the lease row |
| Signed URL → Supabase Storage | 60-second TTL URL; no additional auth on the URL itself (Supabase design) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-19 | Information Disclosure | Tenant accessing another tenant's lease | mitigate | leases_select_tenant RLS binds tenant_id = person_id() — a different tenant's leaseId passed to generateLeaseDownloadUrl returns null |
| T-02-20 | Information Disclosure | Tenant sharing signed URL | accept | 60-second expiry limits window; sharing a document you legitimately have access to is not a security violation |
| T-02-21 | Elevation of Privilege | Tenant navigating to /leases | mitigate | Middleware (from Phase 1) enforces role-based route access; tenant role cannot reach (manager) route group |
| T-02-SC | Tampering | npm installs | accept | No new packages in this plan |
</threat_model>
