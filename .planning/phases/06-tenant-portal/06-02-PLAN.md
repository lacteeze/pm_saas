  <name>Task 1: Server Actions for checklist CRUD</name>
  <files>canary-propos/src/app/(tenant)/my-home/checklist/actions.ts</files>
  <behavior>
    - createChecklist(leaseId, type, title, items: string[]) → inserts checklists row then inserts checklist_items rows in a loop with position index; caller must be manager role; returns checklist id
    - updateChecklistItem(itemId, checked: boolean, note: string | null) → UPDATE checklist_items SET checked=$1, note=$2, checked_at=CASE WHEN $1 THEN now() ELSE NULL END WHERE id=$3; caller must be tenant; checklist must not be submitted
    - submitChecklist(checklistId) → UPDATE checklists SET submitted_at=now(), submitted_by=person.id WHERE id=$1 AND submitted_at IS NULL; also sets checked_at=now() on all items where checked=true and checked_at IS NULL atomically
  </behavior>
  <action>
Create canary-propos/src/app/(tenant)/my-home/checklist/actions.ts as a 'use server' module.

createChecklist: Takes leaseId (UUID), type ('move_in' | 'move_out'), title (string), items (string[] of label text). Auth guard: role.includes('manager') or role.includes('employee') — redirect to /dashboard otherwise. Insert one row into checklists with org_id from JWT claims (get from supabase.auth.getUser() then decode, or fetch from people join org_members). Insert checklist_items rows: map items array to {checklist_id, position: index, label}. Call revalidatePath for the lease detail page.

updateChecklistItem: Takes itemId, checked, note. Auth guard: role.includes('tenant'). UPDATE checklist_items — set checked, note, and checked_at (now() if checked=true, null if false). The RLS policy already blocks updates on submitted checklists; the action does not need a second app-level check but should handle the RLS error gracefully and return {error: 'Checklist already submitted'}.

submitChecklist: Takes checklistId. Auth guard: role.includes('tenant'). Run two statements: (1) UPDATE checklists SET submitted_at=now(), submitted_by=person_id WHERE id=checklistId AND submitted_at IS NULL; (2) UPDATE checklist_items SET checked_at=now() WHERE checklist_id=checklistId AND checked=true AND checked_at IS NULL. If the first UPDATE returns count=0, checklist was already submitted — return error. Call revalidatePath('/my-home/checklist').

org_id resolution: SELECT org_id FROM org_members WHERE user_id = auth.uid() LIMIT 1. This is the established pattern from prior phases.
  </action>
