announcements table:

- title TEXT NOT NULL

RLS: ENABLE ROW LEVEL SECURITY.

Policies:
- "managers full access" — FOR ALL USING (org_id = get_org_id() AND role_is_manager_or_employee()) WITH CHECK (org_id = get_org_id() AND role_is_manager_or_employee())
- "tenants read own property" — FOR SELECT USING (
    property_id IN (
      SELECT u.property_id FROM leases l
      JOIN units u ON u.id = l.unit_id
      WHERE l.tenant_id = get_person_id() AND l.status = 'active'
    )
    AND (expires_at IS NULL OR expires_at > now())
  )

CREATE POLICY "tenants update last_seen" ON people

| tenant → checklist_items UPDATE | Tenant can check items and add notes; must not modify other tenants' items |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-02 | Elevation of Privilege | people UPDATE policy | mitigate | Application server action ONLY sets last_seen_announcements_at; RLS cannot restrict columns but the policy is named and documented so future devs don't widen it |
| T-06-03 | Information Disclosure | announcements SELECT | mitigate | Tenant policy joins through leases → units to derive property_id — no direct property_id claim from JWT |
